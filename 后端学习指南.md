## 第一步：理解项目结构 🏗️

### 1.1 项目分层架构

这个项目使用了经典的三层架构：

```
Controller 层（控制器层）
    ↓ 接收请求
Service 层（业务逻辑层）
    ↓ 处理业务
Mapper 层（数据访问层）
    ↓ 操作数据库
Database（数据库）
```

### 1.2 目录结构详解

```
src/main/java/com/yupi/usercenter/
├── common/              # 通用类
│   ├── BaseResponse.java       # 统一返回结果

├── mapper/              # 数据访问层（操作数据库）

```

### 1.3 每一层的作用

**Controller 层（控制器）**
- 作用：接收前端的 HTTP 请求
- 职责：参数校验、调用 Service、返回结果
- 比喻：就像餐厅的服务员，接收顾客点餐

**Service 层（业务逻辑）**
- 作用：处理具体的业务逻辑
- 职责：实现注册、登录等功能
- 比喻：就像餐厅的厨师，负责做菜

**Mapper 层（数据访问）**
- 作用：与数据库交互
- 职责：增删改查数据
- 比喻：就像餐厅的仓库管理员，负责取食材

## 第二步：学习数据库设计 💾

### 2.1 用户表结构

```sql
CREATE TABLE user (
    id           bigint          # 用户ID（主键）
    username     varchar(256)    # 用户昵称
    userAccount  varchar(256)    # 登录账号
    avatarUrl    varchar(1024)   # 头像地址
    gender       tinyint         # 性别
    userPassword varchar(512)    # 密码（加密后）
    phone        varchar(128)    # 手机号
    email        varchar(512)    # 邮箱
    userStatus   int             # 状态（0-正常）
    createTime   datetime        # 创建时间
    updateTime   datetime        # 更新时间
    isDelete     tinyint         # 是否删除（0-未删除）
    userRole     int             # 角色（0-普通用户，1-管理员）
    planetCode   varchar(512)    # 星球编号
)
```

### 2.2 重要字段说明

- **id**: 唯一标识，自动递增
- **userAccount**: 用来登录的账号
- **userPassword**: 密码（使用 MD5 加密）
- **userRole**: 0 表示普通用户，1 表示管理员
- **isDelete**: 逻辑删除标记（软删除）

## 第三步：核心业务逻辑 🎯

### 3.1 用户注册流程

```
1. 接收注册请求（账号、密码、确认密码、星球编号）
   ↓
2. 参数校验
   - 账号长度 >= 4
   - 密码长度 >= 8
   - 密码和确认密码一致
   - 账号不包含特殊字符
   ↓
3. 检查账号是否已存在
   ↓
4. 密码加密（MD5 + 盐值）
   ↓
5. 保存到数据库
   ↓
6. 返回用户ID
```

### 3.2 用户登录流程

```
1. 接收登录请求（账号、密码）
   ↓
2. 参数校验
   ↓
3. 密码加密
   ↓
4. 查询数据库（账号 + 加密密码）
   ↓
5. 用户脱敏（隐藏敏感信息）
   ↓
6. 保存登录态到 Session
   ↓
7. 返回用户信息
```

## 第四步：API 接口设计 🌐

### 4.1 RESTful API 规范

```
POST   /api/user/register  # 注册
POST   /api/user/login     # 登录
POST   /api/user/logout    # 注销
GET    /api/user/current   # 获取当前用户
GET    /api/user/search    # 搜索用户
POST   /api/user/delete    # 删除用户
```

### 4.2 统一返回格式

```json
{
    "code": 0,              // 状态码（0-成功）
    "data": {...},          // 返回数据
    "message": "ok",        // 消息
    "description": ""       // 详细描述
}
```

### 4.3 错误码设计

```java
SUCCESS(0, "ok", ""),                    // 成功
PARAMS_ERROR(40000, "请求参数错误", ""),  // 参数错误
NULL_ERROR(40001, "请求数据为空", ""),    // 数据为空

NO_AUTH(40101, "无权限", ""),            // 无权限
SYSTEM_ERROR(50000, "系统内部异常", "")  // 系统错误
```

## 第五步：常用技术点 🔧

### 5.1 Spring Boot 注解

```java
@RestController      // 标记这是一个控制器
@RequestMapping      // 映射请求路径
@PostMapping         // 处理 POST 请求
@GetMapping          // 处理 GET 请求
@RequestBody         // 接收 JSON 数据
@Service             // 标记这是一个服务类
@Resource            // 注入依赖
```

### 5.2 MyBatis Plus

```java
// 继承 BaseMapper 自动拥有增删改查方法
public interface UserMapper extends BaseMapper<User> {
}

// 常用方法
save(user)           // 插入
removeById(id)       // 删除
updateById(user)     // 更新
getById(id)          // 查询
list(queryWrapper)   // 列表查询
```

### 5.3 Session 会话管理

```java
// 保存登录态
request.getSession().setAttribute("userLoginState", user);

// 获取登录态
User user = (User) request.getSession().getAttribute("userLoginState");

// 清除登录态
request.getSession().removeAttribute("userLoginState");
```
