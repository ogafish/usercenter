# 🧬 继承（extends）和泛型（<>）详解

## 🎯 你要理解的代码

```java
public interface UserMapper extends BaseMapper<User> {
                              ↑              ↑
                              |              |
                           继承关系        泛型参数
}
```

---

## 📖 第一部分：extends（继承）

### 什么是继承？

**继承** 就是"子类获得父类的所有能力"。

### 生活中的比喻

```
父亲会：
- 做饭
- 开车
- 修电脑

儿子继承父亲：
- 自动会做饭 ✅
- 自动会开车 ✅
- 自动会修电脑 ✅
- 还可以学新技能（打游戏）
```

### Java 中的继承

```java
// 父类（BaseMapper）
public interface BaseMapper<T> {
    int insert(T entity);           // 插入方法
    int deleteById(Long id);        // 删除方法
    int updateById(T entity);       // 更新方法
    T selectById(Long id);          // 查询方法
    List<T> selectList(Wrapper<T> wrapper);  // 查询列表
    // ... 还有 20+ 个方法
}

// 子类（UserMapper）继承父类
public interface UserMapper extends BaseMapper<User> {
    // 什么都不写
    // 但是自动拥有了父类的所有方法！
}

// 使用时
UserMapper userMapper = ...;
userMapper.insert(user);        // ✅ 可以用！来自父类
userMapper.selectById(1L);      // ✅ 可以用！来自父类
userMapper.deleteById(1L);      // ✅ 可以用！来自父类
```

### 继承的好处

```java
// 没有继承：每个 Mapper 都要写一遍
public interface UserMapper {
    int insert(User user);
    int deleteById(Long id);
    User selectById(Long id);
    // ... 写 30 个方法
}

public interface OrderMapper {
    int insert(Order order);
    int deleteById(Long id);
    Order selectById(Long id);
    // ... 又写 30 个方法（重复！）
}

// 有了继承：只需要一行
public interface UserMapper extends BaseMapper<User> {
    // 完成！自动拥有 30 个方法
}

public interface OrderMapper extends BaseMapper<Order> {
    // 完成！自动拥有 30 个方法
}
```

## 📖 第二部分：泛型 <User>

### 什么是泛型？

**泛型** 就是"类型参数"，让代码更灵活、更安全。

### 生活中的比喻

```
通用遥控器：
- 可以控制电视 <电视>
- 可以控制空调 <空调>
- 可以控制音响 <音响>

BaseMapper：
- 可以操作 User 表 <User>
- 可以操作 Order 表 <Order>
- 可以操作 Product 表 <Product>
```

### 泛型的语法

```java
类名<类型参数>

例如：
BaseMapper<User>      // 操作 User 类型
BaseMapper<Order>     // 操作 Order 类型
List<String>          // 存储 String 类型
List<Integer>         // 存储 Integer 类型
```

### 为什么需要泛型？

#### 问题：没有泛型（不安全）

```java
// 没有泛型的 List
List list = new ArrayList();
list.add("字符串");
list.add(123);        // 可以加任何类型
list.add(new User()); // 混乱！

// 取出时需要强制转换
String str = (String) list.get(0);  // 可能出错！
Integer num = (Integer) list.get(1); // 可能出错！
```

#### 解决：使用泛型（安全）

```java
// 有泛型的 List
List<String> list = new ArrayList<>();
list.add("字符串");   // ✅ 可以
list.add(123);        // ❌ 编译错误！只能加 String
list.add(new User()); // ❌ 编译错误！只能加 String

// 取出时不需要转换
String str = list.get(0);  // ✅ 安全！
```

## 🔍 详细解析：BaseMapper<User>

### BaseMapper 的定义

```java
// MyBatis Plus 提供的 BaseMapper
public interface BaseMapper<T> {
                           ↑
                           |
                      泛型参数（类型占位符）
    
    // 所有方法都使用 T 这个类型
    int insert(T entity);              // T 是什么类型？
    int deleteById(Serializable id);
    int updateById(T entity);          // T 是什么类型？
    T selectById(Serializable id);     // 返回 T 类型
    List<T> selectList(Wrapper<T> wrapper);  // 返回 T 的列表
}
```

### UserMapper 继承时指定类型

```java
public interface UserMapper extends BaseMapper<User> {
                                                ↑
                                                |
                                    把 T 替换成 User
}

// 效果：BaseMapper 中所有的 T 都变成了 User
public interface UserMapper extends BaseMapper<User> {
    // 相当于：
    int insert(User entity);              // T → User
    int updateById(User entity);          // T → User
    User selectById(Serializable id);     // T → User
    List<User> selectList(Wrapper<User> wrapper);  // T → User
}
```

### 对比不同的 Mapper

```java
// UserMapper：操作 User 表
public interface UserMapper extends BaseMapper<User> {
    // insert(User user)
    // User selectById(Long id)
}

// OrderMapper：操作 Order 表
public interface OrderMapper extends BaseMapper<Order> {
    // insert(Order order)
    // Order selectById(Long id)
}

// ProductMapper：操作 Product 表
public interface ProductMapper extends BaseMapper<Product> {
    // insert(Product product)
    // Product selectById(Long id)
}
```

## 💡 完整示例

### 1. BaseMapper 的定义（MyBatis Plus 提供）

```java
public interface BaseMapper<T> {
    /**
     * 插入一条记录
     * @param entity 实体对象
     */
    int insert(T entity);
    
    /**
     * 根据 ID 删除
     * @param id 主键ID
     */
    int deleteById(Serializable id);
    
    /**
     * 根据 ID 修改
     * @param entity 实体对象
     */
    int updateById(T entity);
    
    /**
     * 根据 ID 查询
     * @param id 主键ID
     */
    T selectById(Serializable id);
    
    /**
     * 查询列表
     * @param queryWrapper 查询条件
     */
    List<T> selectList(Wrapper<T> queryWrapper);
    
    /**
     * 查询总数
     * @param queryWrapper 查询条件
     */
    Long selectCount(Wrapper<T> queryWrapper);
}
```

### 2. UserMapper 继承并指定类型

```java
public interface UserMapper extends BaseMapper<User> {
    // 继承后，自动拥有：
    // int insert(User entity);
    // int deleteById(Serializable id);
    // int updateById(User entity);
    // User selectById(Serializable id);
    // List<User> selectList(Wrapper<User> queryWrapper);
    // Long selectCount(Wrapper<User> queryWrapper);
}
```

### 3. 实际使用

```java
@Service
public class UserServiceImpl {
    
    @Resource
    private UserMapper userMapper;
    
    public void test() {
        // 1. 插入用户
        User user = new User();
        user.setUsername("张三");
        userMapper.insert(user);  // ← 参数类型是 User
        
        // 2. 查询用户
        User result = userMapper.selectById(1L);  // ← 返回类型是 User
        
        // 3. 更新用户
        user.setUsername("李四");
        userMapper.updateById(user);  // ← 参数类型是 User
        
        // 4. 查询列表
        QueryWrapper<User> wrapper = new QueryWrapper<>();
        wrapper.eq("userStatus", 0);
        List<User> userList = userMapper.selectList(wrapper);  // ← 返回 List<User>
    }
}
```

## 🎓 类型安全的好处

### 没有泛型（不安全）

```java
// 假设没有泛型
public interface UserMapper extends BaseMapper {
    // 方法返回 Object 类型
}

// 使用时
Object obj = userMapper.selectById(1L);
User user = (User) obj;  // ← 需要强制转换，可能出错！

// 如果不小心
Order order = (Order) obj;  // ← 运行时错误！ClassCastException
```

### 有泛型（安全）

```java
// 有泛型
public interface UserMapper extends BaseMapper<User> {
    // 方法返回 User 类型
}

// 使用时
User user = userMapper.selectById(1L);  // ← 不需要转换，类型安全！

// 如果写错
Order order = userMapper.selectById(1L);  // ← 编译错误！立即发现问题
```

```

    | extends
    |

    | 拥有的方法：
    |

    ├─ updateById(User user)
    ├─ User selectById(Long id)

    └─ Long selectCount(Wrapper<User> wrapper)
```

## 🎯 关键理解

### 1. extends 的作用

```
UserMapper extends BaseMapper<User>
           ↑
           |
    继承 = 获得父类的所有方法
```

**效果**：
- UserMapper 自动拥有 BaseMapper 的所有方法
- 不需要自己写这些方法
- 代码更简洁

### 2. <User> 的作用

```
BaseMapper<User>
           ↑
           |
    泛型 = 指定操作的类型
```

**效果**：
- 所有方法的参数和返回值都是 User 类型
- 类型安全，编译时检查
- 不需要强制转换

### 3. 组合起来

```java
public interface UserMapper extends BaseMapper<User> {
                              ↑              ↑
                              |              |
                          继承父类      指定类型为 User
}

// 结果：
// 1. 继承了 BaseMapper 的所有方法
// 2. 所有方法都操作 User 类型
// 3. 类型安全，不会出错
```

```java
// 使用时
userMapper.insert(user);        // ✅ 只能插入 User

userMapper.insert(order);       // ❌ 编译错误！类型不匹配
```
